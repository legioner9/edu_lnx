Экраны мобильных и браузерных приложений


Databag разработан для обеспечения высокой эффективности и потребляет минимальное количество ресурсов хостинга. К числу важных особенностей относятся:

Децентрализованная система (прямая связь между приложением и серверным узлом).
Федеративный режим (учетные записи на разных узлах могут взаимодействовать друг с другом).
Идентификация на основе открытого и закрытого ключей (не привязана к какому-либо блокчейну или домену хостинга)
Сквозное шифрование (администратор хостинга не сможет просматривать темы, если они зашифрованы).
Аудио- и видеозвонки (для обхода NAT требуется отдельный ретрансляционный сервер).
Тематические ветки обсуждений (сообщения организованы по темам, а не по контактам).
Количество участников не ограничено (нет ограничений на количество участников в групповой ветке обсуждения).
Легковесный (сервер может работать на Raspberry Pi Zero версии 1.3)
Низкая задержка (использование веб-сокетов для отправки push-уведомлений во избежание опроса).
Неограниченное количество учетных записей на узел (хост для всей вашей семьи)
Мобильные оповещения о новых контактах, сообщениях и звонках (поддерживает UnifiedPush, FCM, APN)
Многофакторная аутентификация (интегрируется с приложениями TOTP)

  

Приложение доступно на fdroid, а также в магазинах Google и Apple. Вы можете протестировать проект здесь , но не публикуйте ничего важного, так как этот сервер регулярно очищается.

Установка
Для использования Databag вам потребуется DNS-имя, указывающее на ваш узел, с сертификатом. Вы можете развернуть узел вручную, но гораздо проще будет использовать контейнерный сервис. Контейнеры для arm64 и amd64 доступны здесь .

Docker Compose
Запуск контейнера DockerHub с помощью команды `docker compose`:

Стандартный запуск
# From the net/container sub directory:
docker-compose -f compose.yaml -p databag up
Запуск с использованием HTTPS-сертификата Certbot
# FIRST: create a DNS entry in your DNS to point your desired subdomain to your host
# SECOND: edit the net/container/docker-compose-swag.yml to include your domain name
# THIRD: From the root of the project directory:
mkdir -p ~/appdata
docker-compose -f net/container/docker-compose-swag.yml -p databag up
Пример использования Portainer и Nginx Proxy Manager
Из журнала Portainer:

В режиме просмотра громкости нажмите «Добавить громкость»:
Введите имя, затем нажмите «Создать том».
В окне просмотра контейнера нажмите «Добавить контейнер»:
В поле «Изображение» введите «balzack/databag:latest».
Нажмите «Опубликовать новый сетевой порт» и выберите порт 7000 как для хоста, так и для контейнера.
В разделе «Расширенные настройки контейнера» выберите «Env» и нажмите «Добавить переменную среды».
Введите в поле «Имя» значение «АДМИНИСТРАТОР» и ваш пароль администратора [password].
В разделе «Расширенные настройки контейнера» выберите «Тома», затем «Подключить дополнительный том».
Введите '/var/lib/databag' в поле 'контейнер' и название созданного тома в поле 'том'.
Нажмите «Развернуть контейнер».
Из Nginx Proxy Manager:

Добавьте хост и укажите:
Имя хоста [hostname.domain]
IP-адрес Portainer [адрес]
Порт '7000'
Запросить новый SSL-сертификат
Конфигурация прокси Nginx ➡️ Нажмите, чтобы развернуть ⬅️
Из вашего браузера:

Введите адрес вашего сервера в адресную строку [имя_хоста.домен]
Нажмите на значок шестеренки в правом верхнем углу.
Введите пароль администратора: [password]
Нажмите на значок шестеренки на панели управления.
Введите федеративный хост как [имя хоста.домен]
Нажмите «Сохранить».
Нажмите на значок пользователя, чтобы сгенерировать новую ссылку для учетной записи.
Перейдите по ссылке, чтобы создать учетную запись.

Аудио- и видеозвонки
Databag обеспечивает аудио- и видеозвонки и использует ретрансляционный сервер STUN/TURN для обхода NAT. Тестирование проводилось как с coturn , так и с cloudflare , и должно работать с любой реализацией. Инструкции по установке сервера coturn приведены здесь .

Если вы хотите включить аудио- и видеозвонки, вам следует настроить собственный ретрансляционный сервер или использовать сервис Cloudflare TURN . Однако для целей тестирования вы можете использовать демонстрационную конфигурацию ретрансляционного сервера. В модальном окне административной настройки установите:

Включить вызовы WebRTC: -включить-
URL WebRTC-сервера: turn:34.210.172.114:3478?transport=udp
Имя пользователя WebRTC: user
Пароль WebRTC: pass
